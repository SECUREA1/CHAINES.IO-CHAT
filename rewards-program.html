            Leaderboard + profile logic (detailed)

            • Source of truth: every user is stored in chaines_rewards_v1 with points, history, and optional flags.
            • Scoring: earn actions add positive points; redemptions write negative point entries and reduce available balance instantly.
            • Eligibility: leaderboard includes every non-empty username with a valid rewards object.
            • Tie-break: if two users have equal points, names are ordered alphabetically for stable ranking.
            • Profile integration: profile page reads the same rewards store, shows rank, tier progress, recent events, and top leaderboard entries.
            • History cap: rewards history is trimmed to the most recent 120 records to keep storage responsive.
